mpsify

Run CUDA-hardcoded PyTorch code on a Mac (Apple Silicon / MPS) — zero source edits.

pip install mpsify

You inherit a PyTorch training script full of .cuda(), device='cuda', torch.cuda.amp. On a Mac it dies with “Torch not compiled with CUDA enabled” — even though the math would run fine on Apple's Metal (MPS) backend. mpsify patches torch at import time so every CUDA call transparently retargets to MPS. No rewrite from cuda to mps.

# run any CUDA-hardcoded script on your Mac, unchanged
python -m mpsify train.py --epochs 10

What it fixes

  • AssertionError: Torch not compiled with CUDA enabled on Mac
  • RuntimeError: No CUDA GPUs are available
  • Loading a CUDA-trained checkpoint on Apple Silicon (mpsify.load(...))
  • Silent CPU fallbacks — warned live, summarised at exit

Works on

Apple Silicon Macs — M1, M2, M3, M4. torchvision (ResNet, EfficientNet, ViT), timm, and HuggingFace transformers verified numerically identical to CPU.
